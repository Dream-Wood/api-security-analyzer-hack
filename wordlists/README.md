# Wordlists для Endpoint Discovery

Эта директория содержит словари для поиска незадокументированных API эндпоинтов.

## Структура словарей

### versions.txt
Версии API и сегменты верхнего уровня:
- Позиции: 0-1 (корень, первый уровень)
- Приоритет: высокий
- Примеры: v1, v2, api, internal, beta

### controllers.txt
Популярные контроллеры и ресурсы:
- Позиции: 1-3 (средние уровни)
- Приоритет: средний
- Примеры: users, admin, auth, products, orders

### actions.txt
Действия над ресурсами:
- Позиции: глубокие уровни (после ресурсов)
- Приоритет: средний
- Примеры: create, delete, activate, reset, export

### common-endpoints.txt
Часто встречающиеся скрытые эндпоинты:
- Позиции: любые
- Приоритет: высокий
- Примеры: health, debug, admin, swagger, metrics

## Формат файлов

```
# Комментарий (строки начинающиеся с #)
слово1
слово2
слово3

# Пустые строки игнорируются
```

## Создание своих словарей

Вы можете создать свои словари для специфичных проектов:

1. Создайте файл с расширением `.txt`
2. Добавьте слова (по одному на строку)
3. Используйте комментарии для описания

### Именование с метаданными

Формат: `имя_приоритет_позиции.txt`

Примеры:
- `custom_high_0-1.txt` - высокий приоритет, позиции 0 и 1
- `myapi_50_2-3-4.txt` - приоритет 50, позиции 2, 3, 4
- `endpoints.txt` - дефолтные настройки

Приоритеты:
- `high` = 80
- `medium` = 50 (default)
- `low` = 20
- Или числовое значение (0-100)

## Использование

Словари автоматически загружаются из этой директории при запуске discovery.

```bash
# CLI
java -jar cli.jar -m discovery \
  --wordlist-dir ./wordlists \
  -u https://api.example.com \
  spec.yaml

# WebUI
# Выберите Discovery preset в интерфейсе
```

## Советы

1. **Меньше = лучше**: Большие словари медленнее и создают много шума
2. **Специализация**: Создавайте словари под конкретные технологии (Rails, Django, Spring)
3. **Приоритеты**: Используйте высокий приоритет для самых вероятных слов
4. **Позиции**: Ограничивайте позиции для увеличения точности
